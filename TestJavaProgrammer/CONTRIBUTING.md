# Contributing Guidelines

Thank you for your interest in contributing to the project!

## Code of Conduct

By participating in this project, you agree to abide by the maintainers' code of conduct.

## distinct Development Process

1.  **Fork the repository** and create your branch from `main`.
2.  **Clone the repository** locally.
3.  **Create a new branch** for your feature or bugfix:
    ```bash
    git checkout -b feature/amazing-feature
    ```
4.  **Make your changes**. Ensure your code follows the project's coding standards.
5.  **Run tests** to ensure no regressions:
    ```bash
    mvn test
    ```
6.  **Commit your changes** with clear, descriptive commit messages.
7.  **Push to your fork**:
    ```bash
    git push origin feature/amazing-feature
    ```
8.  **Open a Pull Request** targeting the `main` branch.

## Coding Standards

- Follow standard Java CamelCase naming conventions.
- Ensure code is well-documented with Javadoc where appropriate.
- Write unit tests for new features.

## Reporting Bugs

Please use the GitHub Issues page to report bugs. Include:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment details
